# 建筑规范查询系统运行指南

本文档用于指导本系统在本地和局域网环境下的完整运行，包括：
- 环境准备
- Supabase 初始化
- 规范文本导入
- 前后端启动
- 功能验证
- 常见问题排查

## 1. 系统架构

- 前端：`Vue 3 + Vite`（目录：`frontend`）
- 后端：`Node.js + Express`（目录：`backend`）
- 数据库：`Supabase PostgreSQL`
- 核心能力：注册、登录、建筑规范知识库查询

## 2. 环境要求

- Node.js：建议 `v20+`（当前项目已验证 `v24.14.0`）
- npm：随 Node 安装
- Supabase：可用项目 1 个

## 3. 首次部署步骤

### 3.1 初始化 Supabase

1. 创建 Supabase 项目。
2. 打开 Supabase 的 SQL Editor。
3. 执行 `backend/supabase_schema.sql`，创建数据表并写入示例数据。
4. 在项目设置中获取：
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`

### 3.2 配置后端环境变量

1. 复制：`backend/.env.example` -> `backend/.env`
2. 修改为真实值：

```env
PORT=3000
FRONTEND_ORIGIN=*
JWT_SECRET=replace_with_your_jwt_secret
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
```

说明：
- `FRONTEND_ORIGIN=*` 适合内网开发；生产环境建议改成具体域名/IP。
- `JWT_SECRET` 请使用高强度随机字符串。

### 3.3 配置前端环境变量

1. 复制：`frontend/.env.example` -> `frontend/.env`
2. 设置后端地址：

```env
VITE_API_BASE_URL=http://localhost:3000
```

如需局域网访问，请改成运行后端机器的局域网 IP，例如：

```env
VITE_API_BASE_URL=http://192.168.1.20:3000
```

## 4. 启动服务

### 4.1 启动后端

```bash
cd e:\建筑规范查询\backend
npm install
npm run dev
```

后端默认监听：`0.0.0.0:3000`

### 4.2 启动前端

```bash
cd e:\建筑规范查询\frontend
npm install
npm run dev
```

前端默认监听：`0.0.0.0:5173`

## 5. 局域网访问

1. 在服务运行机器执行 `ipconfig`，获取局域网 IP（如 `192.168.1.20`）。
2. 同网段设备访问：
   - 前端：`http://192.168.1.20:5173`
   - 后端 API：`http://192.168.1.20:3000`
3. 若前端不能正常调用接口，检查：
   - `frontend/.env` 中 `VITE_API_BASE_URL` 是否为正确内网 IP
   - 后端是否正在运行
   - Windows 防火墙是否放行 3000/5173 端口

## 6. 规范文本导入（GB 50016 示例）

项目已提供导入脚本：`backend/scripts/import-gb50016.js`

### 6.1 准备文本文件

- 将《建筑设计防火规范》文本整理为 UTF-8 的 `.txt` 文件。
- 建议路径示例：`D:/codes/GB50016-2014-2018.txt`

### 6.2 防重复配置（建议先执行）

在 Supabase SQL Editor 执行：

```sql
delete from public.building_codes a
using public.building_codes b
where a.id > b.id
  and a.code_name = b.code_name
  and coalesce(a.clause_no, '') = coalesce(b.clause_no, '')
  and coalesce(a.title, '') = coalesce(b.title, '');

alter table public.building_codes
add constraint uq_building_codes_code_clause_title
unique (code_name, clause_no, title);
```

### 6.3 执行导入

```bash
cd e:\建筑规范查询\backend
npm run import:gb50016 -- "D:/codes/GB50016-2014-2018.txt" 200
```

参数说明：
- 第 1 个参数：txt 文件路径
- 第 2 个参数：批次大小（可选，默认 200）

### 6.4 导入结果检查

```sql
select count(*) as total
from public.building_codes
where code_name = 'GB 50016-2014（2018年版）';
```

## 7. 功能验证清单

启动后可按以下顺序验证：

1. 打开前端地址。
2. 注册账号并登录。
3. 输入关键词（如“疏散楼梯”“安全出口”）进行查询。
4. 能返回对应规范条目即验证通过。

## 8. 常用接口

- `POST /api/auth/register`：注册
- `POST /api/auth/login`：登录
- `GET /api/knowledge/search?q=关键词`：知识库检索（需 `Bearer token`）
- `GET /api/health`：健康检查

## 9. 常见问题排查

- 后端启动报 `Missing SUPABASE_URL...`
  - 检查 `backend/.env` 是否存在且字段完整。

- 登录成功但查询 401
  - 检查前端是否携带了 `Authorization: Bearer <token>`。

- 查询无结果
  - 先在 Supabase 中确认 `building_codes` 有数据。
  - 检查导入文本格式是否满足脚本解析规则（条号 + 标题 + 内容）。

- 局域网设备打不开页面
  - 确认服务监听地址是 `0.0.0.0`。
  - 检查防火墙端口放行。
  - 确认访问 IP 与服务机器在同一网段。
